## 4️⃣ 🔴 نقاط الصعوبة والمخاطر

### صعوبة عالية 🔴

#### 1. نظام البريد الإلكتروني (المرحلة 9)
```
المشكلة: لا يوجد أي نظام بريد في المشروع
الحل:    تثبيت nodemailer + إعداد SMTP (Brevo/SendGrid/Gmail)
الوقت:   3-4 ساعات إعداد + ساعة لكتابة القالب
التكلفة: Brevo مجاني حتى 300 إيميل/يوم
التعقيد:  ⭐⭐⭐⭐ (إعداد DNS + SPF + DKIM لتجنب Spam)
```

#### 2. الجلسات المؤقتة بدون تسجيل دخول (المرحلة 2-3)
```
المشكلة: النظام الحالي يعتمد JWT + تسجيل دخول إلزامي
         المسار 1 يتطلب "لا تسجيل دخول — session مؤقت"
الحل:    خياران:
         أ) localStorage في المتصفح (أبسط، لكن يضيع مع مسح الكاش)
         ب) إنشاء "مستخدم مجهول" في DB مع UUID (أفضل)
التعقيد:  ⭐⭐⭐⭐ (يتعارض مع منطق auth الموجود)
```

#### 3. محرك الأسئلة الذكية (المرحلة 4)
```
المشكلة: "كل سؤال يعتمد على اللي قبله" = conditional logic
         هذا يحتاج شجرة قرار (Decision Tree) أو state machine
الحل:    كتابة JSON config لشجرة الأسئلة + renderer
التعقيد:  ⭐⭐⭐ (مفهومياً بسيط لكن يحتاج تصميم دقيق)
```

### صعوبة متوسطة 🟡

#### 4. صفحة "لحظة الصدمة" (المرحلة 5)
```
المشكلة: كيف نكتب "الكشف"؟ هل هو نص ثابت أو مولّد ديناميكياً؟
الخيار أ: قوالب نصية مسبقة حسب الإجابات (أبسط وأسرع)
الخيار ب: توليد بالذكاء الاصطناعي (أعقد بكثير)
توصيتي:  ابدأ بخيار أ (قوالب ثابتة) — يمكن ترقيتها لاحقاً
التعقيد:  ⭐⭐⭐
```

## 5️⃣ التغييرات المطلوبة في Prisma Schema

```prisma
// ===== المسار 1: رحلة التشخيص =====

/// جلسة المسار — المستخدم المجهول أو المسجل
model PathSession {
  id          String    @id @default(cuid())
  pathCode    String    // "PATH_1" — للتوسع مستقبلاً
  email       String?   // اختياري — المرحلة 3
  status      String    @default("IN_PROGRESS") // IN_PROGRESS, COMPLETED, ABANDONED
  painChoice  String?   // اختيار الألم — المرحلة 3
  answers     PathAnswer[]
  result      PathResult?
  startedAt   DateTime  @default(now())
  completedAt DateTime?
  lastStepAt  DateTime  @default(now())
  currentStep Int       @default(1)
  
  @@index([email])
  @@index([pathCode])
  @@map("path_sessions")
}

/// إجابات الأسئلة
model PathAnswer {
  id          String      @id @default(cuid())
  sessionId   String
  questionId  String      // "Q1", "Q2", "Q3", "Q4"
  answer      String      // الإجابة (نص أو رقم)
  answeredAt  DateTime    @default(now())
  
  session     PathSession @relation(fields: [sessionId], references: [id], onDelete: Cascade)
  
  @@unique([sessionId, questionId])
  @@map("path_answers")
}

/// نتيجة المسار — الكشف + الرقم
model PathResult {
  id              String      @id @default(cuid())
  sessionId       String      @unique
  diagnosisText   String      // نص الكشف
  singleNumber    Float       // الرقم الواحد
  numberLabel     String      // "ربحك من كل ساعة"
  feedbackChoice  String?     // "نعم" / "جزئياً" / "لا"
  actionStarted   Boolean     @default(false)
  actionData      String?     // JSON — بيانات الخطوة المؤمنة
  
  session         PathSession @relation(fields: [sessionId], references: [id], onDelete: Cascade)
  
  @@map("path_results")
}
```
