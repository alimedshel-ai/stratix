```
│ الرسالة   │ "ركّز على       │ "دعوة للعمق"     │ "وضعك يحتاج      │
│           │  التنفيذ"       │                  │  تفصيلاً"         │
└───────────┴────────────────┴──────────────────┴──────────────────┘
```

### تفاصيل كل مسار:

#### Micro (متناهية الصغر)
```
الشرط:    revenue < 50,000 && employees <= 9
المخرج:   تقرير فوري — لا عمق إضافي
الأقسام:  لقطة مالية + 3 تنبيهات + خطوتين عمل
الرسالة:  "وضعك واضح — ركّز على التنفيذ"
المدة:    3 دقائق فقط
```

#### Small (صغيرة)
```
الشرط:    revenue >= 50,000 && revenue < 500,000 && employees <= 49
المخرج:   تقرير أولي + دعوة للعمق (اختياري)
الأقسام:  لقطة + أنماط + أسئلة عمق
الدعوة:   "لتحليل أعمق (نقطة تعادل دقيقة، سيناريوهات)"
الحافز:   "يستغرق 10 دقائق إضافية"
```

#### Medium (متوسطة)
```
الشرط:    revenue >= 500,000 || employees >= 50
المخرج:   تقرير أولي + عمق موصى به (ضروري)
الأقسام:  لقطة + أنماط متقدمة + فجوات استراتيجية
الدعوة:   "وضعك يحتاج تحليلاً مالياً مفصلاً"
السبب:    "3 مخاطر مالية غير واضحة"
التأكيد:  "ابدأ التحليل العميق — مهم" vs "تخطي على مسؤوليتي"
```

## المرحلة 4: البوابة العميقة (للراغبين)

### Small — عمق معتدل (10 دقائق)

```
القسم 1: هيكل التكاليف (6 حقول — 5 دقائق)
├── تكاليف ثابتة: إيجار، رواتب أساسية، اشتراكات
├── تكاليف متغيرة: بضاعة، عمولات، شحن
└── المخرج: نقطة تعادل دقيقة + الرافعة التشغيلية

القسم 2: تفصيل السيولة (4 حقول — 3 دقائق)
├── آجل مُفصّل: المدة، المبلغ، احتمال التحصيل
├── التزامات عاجلة vs مرنة
└── المخرج: السيولة الحقيقية + دورة التحصيل

القسم 3: سيناريوهات بسيطة (2 سيناريو — 2 دقيقة)
├── سيناريو 1: "ماذا لو رفعت الأسعار 10%؟"
├── سيناريو 2: "ماذا لو خفّضت التكاليف 15%؟"
└── المخرج: أثر مالي واضح + توصية

═══ المخرج النهائي: تقرير مالي مفصل + خطة 6 أشهر ═══
```

### Medium — عمق أكبر (25 دقيقة)

```
القسم 1: محاسبة التكاليف الكاملة (12 حقل)
├── مباشرة: مواد خام، عمالة إنتاج
├── غير مباشرة: إدارة، تسويق، صيانة
└── توزيع: على المنتجات/الخدمات

القسم 2: التنبؤ بالتدفقات (8 حقول — 12 شهر)
├── إيرادات متوقعة شهرياً
├── مصروفات متوقعة
├── التزامات مستقبلية
└── المخرج: رسم بياني 12 شهر + تنبيهات

القسم 3: اقتصاديات الوحدة (6 حقول)
├── تكلفة اكتساب العميل (CAC)
├── عائد العميل الواحد (LTV)
├── هامش المنتج/الخدمة
└── المخرج: هل النمو مربح؟

القسم 4: مختبر القرارات (5 سيناريوهات)
├── توسع: فرع جديد / منتج جديد
├── تقليص: إلغاء خط غير مربح
├── استثمار: تقنية / أتمتة
├── تمويل: قرض vs شريك
├── طوارئ: تراجع 30% في المبيعات
└── المخرج: محاكاة احتمالية (Monte Carlo)

القسم 5: خطة التمويل (4 حقول)
├── المبلغ المطلوب
├── المدة المقترحة
├── المصدر المفضل
└── المخرج: عرض تمويل واضح

═══ المخرج النهائي: نظام مالي متكامل + خطة 12 شهر + عرض تمويل ═══
```

## أنماط الاكتشاف — بالحجم

### أنماط MICRO (1-9 موظفين)

```
🔴 نمط 1: "المالك يفعل كل شي"
   الشرط: employees ≤ 5 + financialSystems < 3
   الوصف: المالك هو المحاسب والمدير والبائع
   الأثر: قرارات عشوائية + إرهاق + خطر توقف
   الحل: محاسب خارجي جزئي (2-4K/شهر)

🔴 نمط 2: "خلط الشخصي بالتجاري"
   الشرط: employees ≤ 5 + لا محاسب
   الوصف: حسابات شخصية = حسابات الشركة
   الأثر: لا يعرف ربحه الحقيقي
   الحل: حساب بنكي منفصل + تطبيق محاسبة بسيط

🟡 نمط 3: "عميل واحد = كل الإيراد"
   الشرط: client_concentration > 50%
   الحل: تنويع قاعدة العملاء قبل أي شي

🟢 نمط 4: "الهامش يكفي للبقاء"
   الشرط: profitMargin > 20%
   السؤال: هل يكفي للنمو؟
```

### أنماط SMALL (10-49 موظف)

```
🔴 نمط 1: "نمو بدون بنية"
   الشرط: revenue > 50K + financialSystems < 3
   الحل: نظام محاسبة قبل أي توسع

🔴 نمط 2: "فجوة تمويل التوسع"
   الشرط: investmentNeeded > monthlyRevenue × 3
   السؤال: ما مصدر التمويل؟

🟡 نمط 3: "تكاليف ثابتة عالية"
   الشرط: expenses/revenue > 75%
   الحل: مراجعة هيكل التكاليف

🟡 نمط 4: "إيرادات غير مستقرة"
   الشرط: revenue_pattern = موسمية أو غير منتظمة
   الحل: احتياطي 3 أشهر + تنويع

🟢 نمط 5: "تمويل ذاتي ممكن"
   الشرط: (profit × 12) >= investmentNeeded
   الجدول: investmentNeeded / monthlyProfit = عدد الأشهر
```

### أنماط MEDIUM (50+ موظف)

```
🔴 نمط 1: "تحصيل آجل خطير"
   الشرط: receivables > revenue × 2
   الأثر: سيولة وهمية

🔴 نمط 2: "اعتماد على خط واحد"
   الشرط: منتج/خدمة واحدة > 70% من الإيراد
   الحل: تنويع مصادر الدخل

🟡 نمط 3: "عدم كفاءة تشغيلية"
   الشرط: expense ratio > 80%
   الحل: إعادة هندسة العمليات
```

## أسئلة العمق — بالحجم

### MICRO (3 أسئلة بسيطة)
```
❓ Q1: "هل تفصل حساباتك الشخصية عن التجارية؟"
   لماذا: يحدد دقة البيانات المالية الأساسية
   أولوية: 🔴 حرج

❓ Q2: "لو توقف أكبر عميل، كم شهر تصمد؟"
   لماذا: يكشف هشاشة الإيراد وقدرة البقاء
   أولوية: 🔴 حرج

❓ Q3: "هل تعرف كم ربحك الحقيقي بعد كل المصاريف؟"
   تلميح: "الربح = ما يبقى بعد كل مصروف + راتبك أنت"
   أولوية: 🟡 مهم
```

### SMALL (5 أسئلة معمقة)
