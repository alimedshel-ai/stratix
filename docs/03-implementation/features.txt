## 🔐 1. Authentication & Authorization

### Pages
- **Login** — Email/password login with demo credential buttons
- **Register** — User registration form

### Features
- JWT token-based authentication
- Role hierarchy: `SUPER_ADMIN > COMPANY_ADMIN > DEPARTMENT_HEAD > DEPARTMENT_USER > VIEWER`
- Role-based sidebar navigation (different menus per role)
- Auth guards on protected routes
- Password hashing (bcrypt)
- Session persistence via localStorage

### API Endpoints
- `POST /auth/login` — Login with email/password → returns JWT token
- `POST /auth/register` — Register new user
- `GET /auth/profile` — Get current user profile

- **Company Edit** — Same form pre-filled

### Features

- `POST /api/companies` — Create

## 🧠 12. Intelligence & Health

### Pages
- **Entity Health** — Health score dashboard (A-F grade)
- **Entity Alerts** — Active alerts list
- **Executive Dashboard** — Executive summary

### Features
- Strategy Health Score calculation (weighted formula):
  - Alignment (30%)
  - Completeness (20%)
  - Performance (40%)
  - Freshness (10%)
- Health grade: A (90+), B (75+), C (60+), D (40+), F (<40)
- Automated alert generation:
  - KPI underperformance alerts
  - Review delay alerts
  - Stalled draft alerts
  - Initiative delay alerts
  - Performance drop alerts
- Smart recommendations (cross-reference KPIs with SWOT analysis)
- Review intelligence (CONTINUE / ADJUST / PIVOT recommendations)

### API Endpoints
- `GET /api/intelligence/health/:entityId` — Health score
- `GET /api/alerts/entity/:entityId` — Entity alerts
- `POST /api/alerts/dismiss/:id` — Dismiss alert
- `GET /api/intelligence/recommendations/:entityId` — Smart recommendations
