## **القسم الأول: المستوى الاستراتيجي**
### (نظرة من أعلى — Birds-eye view)

---

### **المنهجية 1: إدارة التقييمات المبنية على الأداء**

**المستوى:** استراتيجي  
**المسؤول:** الإدارة العليا  
**المدى الزمني:** 3-5 سنوات  
**السؤال:** ماذا نقيس؟ ولماذا؟

**الفلسفة:**
> تحويل النظام من "مفتوح" إلى "حلقة مغلقة"

**المراحل الستة:**

```
المرحلة 1 — التأسيس (شهران):
├── تشكيل فريق متعدد التخصصات
├── دراسة الوضع الحالي (Gap Analysis)
├── اختيار إطار العمل (BSC أو Logic Model)
└── تحديد المؤشرات (SMART-C)

المرحلة 2 — البناء التقني (شهران):
├── تصميم قاعدة البيانات
├── بناء محرك الحساب
├── نظام الإنذار المبكر
└── لوحات القيادة

المرحلة 3 — التطبيق التجريبي (شهران):
├── اختيار قسم واحد
├── تطبيق دورة PDCA
└── جمع الدروس المستفادة

المرحلة 4 — التوسع (6 أشهر):
├── قسم → 3 أقسام → المؤسسة كاملة
└── برنامج تدريب متدرج

المرحلة 5 — الربط بالمنظومة:
├── ربط بالموازنة (Performance-Based)
├── ربط بالموارد البشرية
└── الخيط الذهبي (Golden Thread)

المرحلة 6 — التحسين المستمر:
├── مراجعة يومية للمؤشرات الحرجة
├── أسبوعية، شهرية، ربع سنوية
└── انحراف >10% = أصفر | >25% = أحمر
```

**الأدوات المستخدمة:**
- BSC (Balanced Scorecard)
- Logic Model
- مؤشرات SMART-C

---

### **المنهجية 2: ربط القطاعات بالاستراتيجيات الوطنية**

**المستوى:** استراتيجي  
**المسؤول:** الإدارة العليا + مديرو القطاعات  
**المدى الزمني:** 5-10 سنوات  
**السؤال:** ماذا نحقق؟ ولماذا هذا القطاع مهم؟

**الفلسفة:**
> كل نشاط يرتبط بهدف وطني قابل للقياس

**الهيكل الخماسي:**

```
المستوى 1: القطاع
├── تصنيف ISIC
├── نوع الملكية
└── المساهمة في GDP

المستوى 2: الكيان
├── الرؤية والرسالة
├── الأهداف الاستراتيجية
└── KPIs الرئيسية

المستوى 3: المحفظة/البرنامج
├── مجموعة مشاريع مترابطة
└── ميزانية موحدة

المستوى 4: المشروع
├── بداية ونهاية محددة
├── ميزانية (CAPEX + OPEX)
└── مدير مشروع

المستوى 5: النشاط
├── عمل يومي متكرر
├── مرتبط بـ ISIC
└── قابل للقياس
```

**الخيط الذهبي:**
```
نشاط → مشروع → برنامج → هدف كيان → 
هدف قطاع → هدف وطني (SDG/رؤية 2030)
```

**الأدوات المستخدمة:**
- ISIC Classification
- Golden Thread View
- Strategic Alignment Matrix

---

### **المنهجية 3: مصفوفة الأولويات الذكية**

**المستوى:** استراتيجي  
**المسؤول:** الإدارة العليا + لجنة التقييم  
**المدى الزمني:** سنوي (لاختيار مشاريع السنة)  
**السؤال:** ماذا نختار؟ ولماذا هذا المشروع أولى؟

**الفلسفة:**
> اختيار المشاريع بمعايير موضوعية — لا بالحدس

**الفئات الخمس:**

```
1. المعايير الاستراتيجية (30%):
   ├── SDGs (10%)
   ├── رؤية 2030 (15%)
   └── أهداف الخطة (5%)

2. معايير الأثر (25%):
   ├── المستفيدون (10%)
   ├── العائد (10%)
   └── المجال الجوهري (5%)

3. المعايير التشغيلية (20%):
   ├── التكلفة (7%)
   ├── الوقت (5%)
   ├── الموارد (5%)
   └── البنية التحتية (3%)

4. معايير الاستدامة (15%):
   ├── الاستعجال (8%)
   └── الاستدامة (7%)

5. معايير المخاطر (10%)
```

**آلية العمل:**
```
ورش عمل → تقييم → حساب تلقائي → 
ترتيب → تحليل الهيمنة → تحليل الحساسية
```

**الأدوات المستخدمة:**
- MCDA (Multi-Criteria Decision Analysis)
- Dominance Analysis
- Sensitivity Analysis
- Monte Carlo Simulation

---

### **المنهجية 4: منطق الربط (Linkage Logic)**

**المستوى:** استراتيجي + تكتيكي (عابر للمستويات)  
**المسؤول:** فريق متعدد التخصصات  
**المدى الزمني:** 2-3 أشهر للبناء  
**السؤال:** كيف يعمل كل شيء معاً؟

**الفلسفة:**
> النظام = جهاز عصبي واحد — لا أدوات منفصلة

**المراحل الثمانية:**

```
1. فهم فلسفة الربط (يوم)
2. تصميم الحلقة المغلقة (أسبوع)
3. ربط SWOT بـ TOWS (أسبوع)
4. بناء الربط السببي BSC (أسبوعان)
5. ربط الموارد البشرية (أسبوع)
6. التشغيل البيني للبيانات (أسبوعان)
7. بناء الجهاز العصبي (أسبوع)
8. الاختبار والمعايرة (أسبوع)
```

**الأدوات المستخدمة:**
- SWOT → TOWS Matrix
- BSC Causal Chain
- Data Dictionaries
- API Integration
- Event-Action Matrix

### **المنهجية 5: من الألم للوضوح (Stratix)**

**المستوى:** تكتيكي  
**المسؤول:** صاحب العمل + الفريق  
**المدى الزمني:** 90 يوم  
**السؤال:** كيف نبدأ؟

**الفلسفة:**
> يسمّي ما تشعر به قبل أن تعرف كيف تقوله

**المراحل الخمسة:**

```
1. الألم (دقيقتان)
2. التعمق الذكي (5 دقائق)
3. الكشف (دقيقة)
4. الخطوة المؤمنة (دقيقة)
5. المتابعة الذكية (7-30 يوم)
```

**الأدوات المستخدمة:**
- سؤال الألم (Pain Point Discovery)
- الكشف (Mirror Technique)
- الرقم الواحد (Single Metric)

---

### **المنهجية 6: الأنماط الستة للشركات**

**المستوى:** تكتيكي  
**المسؤول:** صاحب العمل  
**المدى الزمني:** فوري (التشخيص) + 90 يوم (التنفيذ)  
**السؤال:** ما نمطي؟ وكيف أتقدم؟

**الأنماط:**
```
1. المنهكة → حدد الـ 20% الذهبية
2. الوهمية → احسب هامش الربح
3. الخائفة → فوّض قراراً واحداً
4. المشتتة → اسأل عملاءك الأوفياء
5. الهشة → اكتب القرار المؤجل
6. الجاهزة → عرّف النجاح للفريق
```

**الأدوات المستخدمة:**
- Pattern Recognition
- Customized Action Plans
- Progress Tracking

---

### **المنهجية 7: بناء المنتج MVP**

**المستوى:** تكتيكي  
**المسؤول:** فريق المنتج  
**المدى الزمني:** 6-8 أسابيع  
**السؤال:** كيف نختبر بسرعة؟

**الفلسفة:**
> مسار واحد حتى الكمال → ثم التكرار السريع

**الخطوات:**
```
أسبوع 1-2: بناء المسار 1
أسبوع 3: اختبار 10 شركات
أسبوع 4: تحسين
أسبوع 5-6: إضافة المسارات 2-6
أسبوع 7+: توسع
```

**الأدوات المستخدمة:**
- Lean Startup
- A/B Testing
- User Feedback Loop

---

### **المنهجية 8: هيكل التسعير الثلاثي**

**المستوى:** تكتيكي  
**المسؤول:** فريق المنتج + المبيعات  
**المدى الزمني:** مستمر  
**السؤال:** كيف نُسعّر القيمة؟

**المستويات:**
```
مجاني → الكشف + الخطوة الأولى
Growth → 90 يوم + ضمان
Enterprise → شراكة كاملة
```

**الأدوات المستخدمة:**
- Value-Based Pricing
- Tiered Model
- Upgrade Triggers

---

### **المنهجية 9: الضمان القابل للقياس**

**المستوى:** تكتيكي  
**المسؤول:** فريق النجاح  
**المدى الزمني:** 90 يوم  
**السؤال:** كيف نضمن النتيجة؟

**الشروط:**
```
1. النتيجة قابلة للقياس
2. العميل نفّذ الخطوات
3. المدة 90 يوماً
```

**الأدوات المستخدمة:**
- Success Metrics
- Execution Tracking
- Outcome Measurement

### **المنهجية 10: بناء المسار 1 (14 يوم)**

**المستوى:** تشغيلي  
**المسؤول:** فريق التطوير  
**المدى الزمني:** يومي/أسبوعي  
**السؤال:** متى ننتهي؟ وكيف ننفذ اليوم؟

**الفلسفة:**
> اختبر بسرعة — حسّن بالواقع — كرر

**الجدول الزمني:**
```
اليوم 1: التحضير
اليوم 2: صفحة الدخول
اليوم 3: سؤال الألم
اليوم 4-5: الأسئلة الأربعة
اليوم 6: الكشف + لحظة الصدمة
اليوم 7: الرقم الواحد
اليوم 8: الخطوة المؤمنة
اليوم 9-10: لوحة التقدم
اليوم 11: المتابعة
اليوم 12-14: الاختبار
```

**الأدوات المستخدمة:**
- Kanban Board (لتتبع المهام اليومية)
- Gantt Chart (للجدول الزمني)
- Daily Standup (للمتابعة)
- KPIs التشغيلية (معدل الإكمال، الأخطاء، السرعة)
