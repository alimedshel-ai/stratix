## المرحلة 1: تجربة المستخدم والانطباع الأول
### ⏱️ المدة: 3-4 ساعات | 🎯 الهدف: WOW Factor

> **الفلسفة:** المستخدم يقرر في أول 30 ثانية — لازم ينبهر.

| # | الميزة | الوصف | الجهد | الأولوية |
|---|--------|-------|-------|----------|
| 1.1 | **داشبورد تنفيذي CEO** | صفحة واحدة للمدير: نبض استراتيجي + أخطر 3 مؤشرات + 3 قرارات مطلوبة + رسم بياني Timeline | 2 ساعة | 🔴 حرج |
| 1.2 | **نظام الإنجازات (Gamification)** | شارات وإنجازات: أول هدف، محلل محترف، منظمة منضبطة — شريط "مستوى النضج الاستراتيجي" | 1.5 ساعة | 🔴 حرج |
| 1.3 | **مركز الإشعارات** | جرس في الهيدر: KPI نزل، مهمة متأخرة، مراجعة قادمة — مع عداد أحمر | 1 ساعة | 🟡 عالي |

### 📦 المخرجات:
```
✅ /ceo-dashboard.html   — داشبورد تنفيذي
✅ /js/achievements.js   — محرك الإنجازات
✅ /js/notifications.js  — مركز الإشعارات
✅ تحديث sidebar.js     — إضافة الروابط الجديدة
```

### 🎯 معيار النجاح:
- المستخدم يدخل ← يشوف نبض منظمته فوراً
- شارة "أول خطوة" تظهر بعد إنشاء أول هدف
- الإشعارات تظهر بدون تحديث الصفحة

## 🏗️ المتطلبات التقنية لكل مرحلة

### المرحلة 1 — لا تحتاج تعديل backend
```
Frontend فقط: HTML + CSS + JS
localStorage للإنجازات
API calls موجودة أصلاً
```

### المرحلة 2 — تحتاج schema + routes جديدة
```
prisma: Comment, Activity models
routes: comments.js, activities.js
Frontend: calendar library (vanilla)
```

### المرحلة 3 — تحتاج مكتبات
```
PDF: html2pdf.js أو jsPDF
Charts: Chart.js (موجود)
Risk model في schema
```

### المرحلة 4 — تحتاج infrastructure
```
PWA: manifest.json + service-worker.js
WebSocket: للـ live board (اختياري)
Share links: public tokens
```

### المرحلة 5 — تحتاج AI API
```
OpenAI API أو Claude API
Prompt engineering
Data aggregation layer
```
