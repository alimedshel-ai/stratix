## 3. 🔧 خطة ربط المخاطر بأصحاب المصلحة

### 3.1 الخطوة 1: إنشاء نموذج Stakeholder في Prisma

```prisma
// === NEW: أصحاب المصلحة ===
model Stakeholder {
  id              String    @id @default(cuid())
  entityId        String
  versionId       String?
  
  // البيانات الأساسية
  name            String              // الاسم (مجلس الإدارة، الموظفون...)
  role            String?             // الدور/العلاقة
  category        String?             // INTERNAL, EXTERNAL, MIXED
  
  // مصفوفة القوة/الاهتمام
  power           Int       @default(3)   // 1-5
  interest        Int       @default(3)   // 1-5
  strategy        String    @default("inform")  // manage, satisfy, inform, monitor
  
  // المشاعر والمشاركة
  sentiment       String    @default("NEUTRAL")  // POSITIVE, NEUTRAL, NEGATIVE
  engagement      Int       @default(50)  // 0-100
  
  // التواصل
  contactInfo     String?   // بيانات التواصل
  lastContactAt   DateTime?
  meetingsCount   Int       @default(0)
  
  // ربط المخاطر
  risks           StakeholderRisk[]
  
  // Metadata
  color           String?
  createdBy       String?
  createdAt       DateTime  @default(now())
  updatedAt       DateTime  @updatedAt
  
  entity          Entity    @relation(fields: [entityId], references: [id], onDelete: Cascade)
  version         StrategyVersion? @relation(fields: [versionId], references: [id])
  
  @@index([entityId])
  @@index([versionId])
  @@map("stakeholders")
}

// === NEW: جدول الربط بين المخاطر وأصحاب المصلحة ===
model StakeholderRisk {
  id              String    @id @default(cuid())
  stakeholderId   String
  riskId          String
  impactLevel     String    @default("MEDIUM")  // HIGH, MEDIUM, LOW
  impactType      String?   // AFFECTED_BY, CAUSES, MITIGATES
  notes           String?
  createdAt       DateTime  @default(now())
  
  stakeholder     Stakeholder   @relation(fields: [stakeholderId], references: [id], onDelete: Cascade)
  risk            StrategicRisk @relation(fields: [riskId], references: [id], onDelete: Cascade)
  
  @@unique([stakeholderId, riskId])
  @@index([stakeholderId])
  @@index([riskId])
  @@map("stakeholder_risks")
}
```

### 3.2 الخطوة 2: إنشاء Backend Routes

#### `routes/stakeholders.js`
```
GET    /api/stakeholders                    → قائمة أصحاب المصلحة (مع فلترة)
GET    /api/stakeholders/:id                → تفاصيل صاحب مصلحة + مخاطره
POST   /api/stakeholders                    → إنشاء صاحب مصلحة
PUT    /api/stakeholders/:id                → تعديل صاحب مصلحة
DELETE /api/stakeholders/:id                → حذف صاحب مصلحة
POST   /api/stakeholders/:id/risks          → ربط خطر بصاحب مصلحة
DELETE /api/stakeholders/:id/risks/:riskId  → فك ربط
GET    /api/stakeholders/matrix             → بيانات المصفوفة
```

#### `routes/risks.js`
```
GET    /api/risks                           → قائمة المخاطر (مع فلترة)
GET    /api/risks/:id                       → تفاصيل خطر + أصحاب المصلحة
POST   /api/risks                           → إنشاء خطر
PUT    /api/risks/:id                       → تعديل خطر
DELETE /api/risks/:id                       → حذف خطر
GET    /api/risks/heatmap                   → بيانات الخريطة الحرارية
GET    /api/risks/by-stakeholder/:id        → مخاطر صاحب مصلحة معين
POST   /api/risks/:id/stakeholders          → ربط أصحاب مصلحة بخطر
```
